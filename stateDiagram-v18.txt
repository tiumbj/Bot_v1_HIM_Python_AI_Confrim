===============================================================================
A) PROJECT PURPOSE (ภาพรวม)
===============================================================================
HIM คือระบบเทรดแบบ Hybrid:
1) Engine (Python) คิดแผนเทรดแบบ deterministic: structure/entry/sl/tp/rr/confidence
2) AI เป็นชั้น “confirm-only” (ไม่ตัดสินแทนระบบ) เพื่อเพิ่มความแม่นยำ/ลด noise
3) Validator เป็น gate แบบ fail-closed (ไม่ผ่าน = ห้าม execute)
4) MT5 execution ทำงานอัตโนมัติเมื่อ enable_execution=true
5) Intelligent Dashboard ใช้ “ปรับ config + แสดงข้อมูลช่วยตัดสินใจ manual” โดย auto bot ยัง run ต่อ

จุดเน้น: Production-grade, deterministic, guarded, audit-friendly.

Risk / Threshold (Breakout preset):
- confidence_threshold = 65

- min_score = 6
  เหตุผล: คุมคุณภาพสัญญาณ (คะแนนต่ำกว่านี้ถือว่า noise มาก)
- min_rr = 1.9
  เหตุผล: บังคับ RR เกือบ 2:1 เพื่อชดเชย win-rate ที่อาจแกว่งช่วง breakout
- lot = 0.01
  เหตุผล: lot เริ่มต้นเล็กเพื่อ commissioning/ลด risk ระหว่างปรับระบบ

Indicators parameters (ค่า default ที่เห็นใน manual editor):
- adx_period = 14, adx_max = 35
  เหตุผล: 14 เป็นค่า market standard; adx_max ใช้คุม “แรงเกิน/สวิงสูง” ในบางโหมด
- bb_period = 20, bb_std = 2, bb_width_atr_max = 6
  เหตุผล: BB(20,2) เป็นค่า standard; width/ATR ใช้คุมสภาวะ volatility
- rsi_period = 14, rsi_overbought = 70, rsi_oversold = 30, rsi_soft_band = 25
  เหตุผล: RSI(14) standard; soft_band ใช้กรองสัญญาณใกล้ขอบแบบไม่แข็งเกิน
- supertrend.period = 10, supertrend.mult = 3
  เหตุผล: set มาตรฐานเพื่อช่วยทิศทาง (directional filter)
- structure_sensitivity: htf=5, mtf=4, ltf=3
  เหตุผล: HTF เข้มกว่าเพื่อกัน noise; LTF ต่ำกว่ารับการเข้าเร็ว
- timeframes: htf="H1", mtf="M15", ltf="M5"
  เหตุผล: โครง MTF มาตรฐานของระบบนี้ (bias จาก H1, setup จาก M15, entry จาก M5)

Telegram:

C) TECHNICAL INDICATORS & TOOLS ที่ใช้
===============================================================================
Indicators (ใช้ใน engine/filters):
- ATR (Average True Range): วัด volatility, ใช้เป็นหน่วยระยะ SL/TP, proximity, buffers
- ADX: วัดความแข็งแรงของแนวโน้ม (trend strength), ใช้ filter/โหมด
- Bollinger Band Width (BBWidth): วัดการขยายตัวของความผันผวน
- Market Structure (HH/HL, LH/LL): ระบุ trend
- BOS/CHoCH: สัญญาณยืนยัน/เตือนการเปลี่ยนโครงสร้าง
Tools/Infra:
- MetaTrader5 (mt5): ดึงข้อมูลราคา/ส่งคำสั่ง
- Flask API server: /api/...
- Dashboard HTML/JS: UI ควบคุม/มอนิเตอร์
- GitHub repo: versioning + handoff
- Validator v1.0: schema checking + constraint enforcement
- (Web direct) External status: OpenAI status + reachability checks (UX)

Key config fields ที่ Dashboard ต้องเห็นเป็น “ตัวเลข” (ไม่เป็น null):
- mode
- confidence_threshold
- min_score
- min_rr
- lot (หรือ execution.volume)
- enable_execution (DRY_RUN/LIVE)

สถานะล่าสุดจาก /api/status:
- MT5 account OK: login=168021026, server=XMGlobal-MT5 2, positions_count=0
- config numbers ยังเป็น null (confidence_threshold/min_score/min_rr/lot/mode)
  สาเหตุเชิงระบบ:
  - resolve_effective_config() ไม่เติม default หรือ
  - config.json ไม่มี keys ที่ endpoint คาดหวังในระดับ top-level
